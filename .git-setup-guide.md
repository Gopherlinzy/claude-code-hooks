# 多账户 Git 配置指南

## 概述

本项目维护两个 Git 仓库：
- **GitHub**: `git@github.com:Gopherlinzy/claude-code-hooks.git`（origin）
- **GitLab**: `git@gitcn.yostar.net:sunluyi/claude-code-hooks.git`（gitlab）

需要为两个账户分别配置 `user.name` 和 `user.email`。

## 当前配置状态

```bash
# 查看远程列表
git remote -v

# 查看项目级 git config
git config --list | grep user
```

## 方案 1：自动账户切换（推荐）

### 步骤 1：配置 SSH Key

编辑 `~/.ssh/config` 添加以下配置：

```
# GitHub
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_github
    AddKeysToAgent yes

# GitLab (Yostar)
Host gitcn.yostar.net
    HostName gitcn.yostar.net
    User git
    IdentityFile ~/.ssh/id_ed25519_gitlab
    AddKeysToAgent yes
```

### 步骤 2：生成/配置 SSH Key

如果还没有对应的 SSH key，生成它们：

```bash
# 为 GitHub 生成 key
ssh-keygen -t ed25519 -C "huzohulinyoujia@gmail.com" -f ~/.ssh/id_ed25519_github

# 为 GitLab 生成 key
ssh-keygen -t ed25519 -C "sunluyi@yostar.net" -f ~/.ssh/id_ed25519_gitlab

# 或使用 RSA（较旧系统兼容性更好）
ssh-keygen -t rsa -b 4096 -C "huzohulinyoujia@gmail.com" -f ~/.ssh/id_rsa_github
ssh-keygen -t rsa -b 4096 -C "sunluyi@yostar.net" -f ~/.ssh/id_rsa_gitlab
```

### 步骤 3：添加 Public Key 到账户

```bash
# GitHub
pbcopy < ~/.ssh/id_ed25519_github.pub
# 或
cat ~/.ssh/id_ed25519_github.pub

# 粘贴到 GitHub 账户设置 → SSH Keys

# GitLab
pbcopy < ~/.ssh/id_ed25519_gitlab.pub
# 或
cat ~/.ssh/id_ed25519_gitlab.pub

# 粘贴到 GitLab 账户设置 → SSH Keys
```

### 步骤 4：测试连接

```bash
# 测试 GitHub
ssh -T git@github.com

# 测试 GitLab
ssh -T git@gitcn.yostar.net
```

### 步骤 5：使用自动切换脚本

```bash
# 查看所有配置的账户
./.git-multi-account show

# 自动检测并切换（根据当前追踪的远程）
./.git-multi-account auto

# 手动切换到特定账户
./.git-multi-account set github.com
./.git-multi-account set gitcn.yostar.net
```

## 方案 2：手动配置每个远程

### 针对 GitHub (origin)

```bash
git config --local user.name "linzy"
git config --local user.email "huzohulinyoujia@gmail.com"
```

### 针对 GitLab (gitlab)

```bash
git config --local user.name "sunluyi"
git config --local user.email "sunluyi@yostar.net"
```

> 这种方案的局限：同一项目只能有一个 git config user.name，所以适合单一账户提交。

## 方案 3：使用 Git Hook（自动化）

如果需要完全自动化，可以在 `.git/hooks/pre-commit` 添加：

```bash
#!/bin/bash

# 获取当前分支的远程跟踪信息
TRACKING_BRANCH=$(git rev-parse --abbrev-ref --symbolic-full-name @{u} 2>/dev/null)
REMOTE=$(echo "$TRACKING_BRANCH" | cut -d/ -f1)

case "$REMOTE" in
    origin)
        git config user.name "linzy"
        git config user.email "huzohulinyoujia@gmail.com"
        ;;
    gitlab)
        git config user.name "sunluyi"
        git config user.email "sunluyi@yostar.net"
        ;;
esac
```

## 推送到不同远程的方法

### 推送到 GitHub（origin）

```bash
git push origin main
```

### 推送到 GitLab（gitlab）

```bash
git push gitlab main
```

### 同时推送到两个远程

```bash
# 方法 1：分别推送
git push origin main
git push gitlab main

# 方法 2：配置 push 多个 URL
git remote set-url --add --push origin git@gitcn.yostar.net:sunluyi/claude-code-hooks.git
git push origin main  # 会推送到两个地方

# 查看配置
git remote get-url --push origin
```

### 从两个远程拉取更新

```bash
# 从 GitHub 拉取
git fetch origin

# 从 GitLab 拉取
git fetch gitlab

# 拉取所有远程
git fetch --all
```

## 常见问题

### Q: 我该用哪个账户提交？
A: 如果两个账户都需要主动维护，建议使用自动切换脚本 `.git-multi-account auto`，它会根据当前分支的远程跟踪信息自动选择。

### Q: 如何同步两个仓库?
A: 
```bash
# 从 GitHub 拉取最新
git fetch origin
git merge origin/main

# 推送到 GitLab
git push gitlab main
```

### Q: SSH 连接失败怎么办?
A: 
```bash
# 测试连接
ssh -v git@github.com
ssh -v git@gitcn.yostar.net

# 检查 SSH 配置
cat ~/.ssh/config

# 检查 key 权限
ls -la ~/.ssh/id_*

# key 权限应该是 600
chmod 600 ~/.ssh/id_*
```
